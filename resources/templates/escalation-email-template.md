# Escalation Email Template

Use when escalating a ticket to Tier 2, a vendor, or another team via email.

---

## Template

```
Subject: [ESCALATION] Ticket #[TICKET_ID] – [Brief Issue Description]

Hi [Tier 2 / Team Name],

I'm escalating the following ticket for your assistance. Please see the summary below.

---

Ticket #:       [TICKET_ID]
Priority:       [Low / Medium / High / Critical]
User:           [Full Name] – [Department]
Device:         [Computer Name / Asset Tag]
Reported:       [Date & Time]

---

ISSUE SUMMARY
[1–3 sentences describing what the user is experiencing. Be specific — include
exact error messages, what the user was doing when it happened, and frequency.]

STEPS ALREADY TAKEN
1. [Step 1 and result]
2. [Step 2 and result]
3. [Step 3 and result]

CURRENT STATUS
[What is happening right now? Is the user able to work? Is there a workaround?]

SUSPECTED CAUSE
[Your best theory on the root cause, if any.]

ACTION NEEDED
[Specifically what you need Tier 2 to do or investigate.]

---

User contact: [Email / Phone / Preferred method]
Availability:  [User's available hours if relevant]

Please let me know if you need any additional information.

Thanks,
[Your Name]
[Your Title]
[Help Desk Contact Info]
```

---

## Tips

- Always include the exact error message in quotes — it helps Tier 2 search their own KB
- Don't say "I tried everything" — list what you actually tried
- Set the right expectation with the user: tell them who is handling it next and when they can expect an update
