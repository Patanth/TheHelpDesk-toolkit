# New Ticket Checklist

Use this at the start of every ticket to make sure you gather the right information upfront. Getting this right saves time for everyone.

---

## Intake

- [ ] User's full name
- [ ] Username / employee ID
- [ ] Department / location
- [ ] Contact number or preferred callback method
- [ ] Device type (laptop, desktop, phone)
- [ ] Computer name / asset tag
- [ ] Operating system and version (if relevant)

---

## Issue Details

- [ ] What exactly is the issue? (Let the user describe it first)
- [ ] What error message are they seeing? (Get the exact wording)
- [ ] When did it start?
- [ ] Is it happening every time or intermittently?
- [ ] Has it worked before?
- [ ] What changed recently? (updates, new software, moved offices, password change)
- [ ] Is anyone else affected?

---

## Impact

- [ ] How many users are affected?
- [ ] Is this blocking the user from doing their job?
- [ ] Is there a workaround in place?
- [ ] Is there a deadline or urgency? (payroll, meeting, client-facing)

---

## Ticket Setup

- [ ] Priority set correctly based on impact
- [ ] Category and subcategory assigned
- [ ] Asset linked to ticket
- [ ] Initial notes written before troubleshooting starts

---

> **Tip:** Ask open questions first ("Tell me what's happening") before closed ones ("Is the light on?"). Users often leave out important context when asked yes/no questions.
